

1. 人工智能发展的三个阶段？
1.1 三阶段？
	大模型---工作流---智能体
	llm---workflow---agent
1.2 第一阶段：llm？
	缺点：有个巨大的局限性，就是其知识都是开源的，他没有办法来访问私人数据
1.3 第二阶段：workflow？
	优点：解决了这个局限性，就是通过人工设置任务流程，大模型成为这个流程的一部分
	缺点：流程太过死板和固定，每次有新需求都需要人工调整，不易扩展，适合流程固定的行业
	热点名词：rag就是一种ai workflow
1.4 第三阶段：agent？
	优点：人--》机器。整个任务流程的设计也变成机器了，人彻底退出工作，变成了一个纯需求提出者
	热点名词：react framework。react = reasoning（人推理变成机器推理） + act

===智能体课程
1. 如何制作 agent？
	a）提示词
		agent提示词：提示词里面定义了agent的身份，以及他的响应和输出风格。所以说提示词的好坏影响agent的好坏
					四要素：角色身份，专业技能，输出格式，合法合规
		用户提示词：也就是用户的输入
	b）知识库 / RAG
		RAG：就是用户问题先搜索知识库，检索结果拼接用户问题，再来提问大模型
		RAG知识库的三种类型：文档、表格、图片
		ps：普通大模型，输入直接就是用户查询
			rag大模型，输入是用户查询+检索上下文
	c）函数调用 / function call / api / 插件
		function call：是2023年，openai最先发布的，能够让大模型可以通过api来调用外部函数，增强模型能力
		使用：用户问题--》模型判断是否需要调用外部函数--〉函数结果返回给大模型--》大模型组装输出
	b+c）b和c的区别？
		知识库和插件都是用来解决模型自身能力不足的问题，但是知识库主要补充的是静态的知识，而插件更多是动态的能力
		比如说知识库里面主要存放的是某个领域的专业知识，这些很难通过模型给出
		插件呢是一些复杂的代码调用，可以帮助模型来完成很多复杂功能
	d）工作流
		对话流：交互式，用户与agent不断问答。可能涉及到逻辑判断等等，根据用户输入的不同。常见比如客服agent
		工作流：一次性的，用户给出输入，经过工作流后得到结果
	e）部署
		智能体可以发布到公众号或者coze商店
2. 案例？
	案例1:高情商职场回复助手？
		a）设计提示词以及让模型优化
		b）选择模型
	案例2:英雄联盟游戏助手
		a）制作知识库，选择层级分段，这样来划分检索的段落
		b）制作智能体：选择模型，制作提示词，重点是引入知识库
	案例3:旅游规划助手
		a）选择插件
		b）agent的身份设定提示词里面，需要提前写好 “什么情况下” 调用 “什么插件”
	案例4:天气插件
		a）重点：插件一定要定义好输入和输出参数是什么
	案例5:跨境电商助手
		a）建立工作流
		b）数据的三种流向：一是代码规则直接进行回答；二是知识库RAG回答；三是模型闲聊回答
		c）三大核心节点：插件，知识库，大模型
	案例6:历史人物视频生成
		a）文案生成
		b）图片生成
		c）视频生成
总结：如何做一个agent？
	a）任务的解耦，把复杂的任务拆解成一个一个的任务节点
	b）确定性和非确定性的拆解
		大模型的输出是不确定的
		知识库，代码/api/插件/function call的输出是确定的
	c）数据流的控制
		工作流的本质是数据的流动，所以对于节点直接数据如何传递也是一个设计的重点
	d）agent是什么？
		agent就是以大模型为基座，然后可以塞入各种知识库、插件来丰富它的能力，从而使它可以完成很多复杂任务
